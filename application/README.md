# Application

This folder contains documentation related to the Customer Order Management System application setup, structure, and testing environment.

## Purpose

The application folder documents how the test application is used as the foundation for the QA portfolio project.

This application supports:

- Manual UI testing
- SQL data validation
- API testing with Postman
- Selenium automation
- Smoke testing
- Regression testing
- Defect investigation

## Application Modules

- Login
- Customers
- Products
- Orders
- Reports

## Technology Stack

- Python Flask
- MySQL
- HTML/CSS
- REST API endpoints

## QA Usage

This application provides a realistic environment to demonstrate testing across the full application stack:

UI → Backend → API → Database
